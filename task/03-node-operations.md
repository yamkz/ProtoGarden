# TASK: ノード操作拡張（コピペ・ロック・グループ化）
日時: 2026-03-17

## ROLE
あなたはElectronアプリ開発に精通したシニアフロントエンドエンジニアとしてふるまってください。Figma等のキャンバスエディタのグループ化・ロック機能の実装パターンに詳しく、直感的な操作体験を重視します。

## CORE GOAL
- ワークスペース間でのノードコピー&ペースト機能を実現する
- ノードのロック/アンロック機能で誤操作を防止する
- ノードのグループ化/解除機能で複数ノードをまとまりとして扱えるようにする

## PURPOSE
- 現状、ノードのコピー&ペーストは同一ワークスペース内のみ。別ワークスペースで作った配置を再利用したい
- ノードの配置を確定した後に誤って動かしてしまうことがある。ロック機能で意図しない編集を防ぎたい
- 関連するノード群（例: ヘッダー+コンテンツ+フッター）を1つのまとまりとして扱い、まとめて移動・リサイズしたい

## HARD REQUIREMENTS
- ワークスペース間コピー&ペーストはアセット（画像・HTMLファイル）も含めて複製すること
- ロック中のノードは移動・リサイズ・削除・テキスト編集の全操作をブロックすること
- ロック状態は鍵アイコンで視覚的に示し、ぱっと見でロック状態がわかること
- ロック/アンロックのショートカット: Cmd+L
- グループ化はCmd+G、グループ解除はCmd+Shift+G
- グループは1つの枠で囲まれる表示にすること
- グループのz-indexはグループ全体が1レイヤーとして動くが、内部的にはノード間の階層を維持すること
- グループは1階層のみ（ネスト不可）
- グループ内ノードの個別編集はダブルクリックでグループ内に入る（Figma方式）

## DELIVERABLE
- コピー&ペースト（ワークスペース間対応）、ロック/アンロック、グループ化/解除を実装したProtoGardenの更新版

## DELIVERABLE SPEC

### A) ワークスペース間コピー&ペースト
- Cmd+Cで選択ノードをクリップボードにコピー（既存のワークスペース内コピーを拡張）
- Cmd+Vで貼り付け時、コピー元と異なるワークスペースの場合、アセットファイル（画像・HTML・スナップショット）をコピー先ワークスペースのassets/に複製
- メインプロセス側にアセットコピー用のIPCチャネルを追加
- コピー元ワークスペースIDをクリップボードデータに含めて判定

### B) ロック/アンロック
- 各ノードデータに`locked`プロパティ（boolean）を追加
- ロック中のノードは移動・リサイズ・削除・テキスト編集を全てブロック
- ロック状態の表示: ノード上にさりげない鍵アイコンを表示し、ノード全体を少し区別できる見た目にする
- ショートカット: Cmd+L で選択ノードのロック/アンロックをトグル
- 右クリックメニューにも「ロック」「アンロック」項目を追加
- ロック状態はワークスペースJSONに保存・復元

### C) グループ化/解除
- 複数ノードを選択してCmd+Gでグループ化
- グループは新しいノードタイプ「group」として管理。子ノードIDのリストを保持
- グループ表示: 子ノード群を囲む枠（薄いボーダー）を表示
- グループのドラッグ: 子ノード全体が一緒に移動
- グループのリサイズ: 子ノード全体が比率を保ってスケール
- グループのz-index: グループ全体が1レイヤーとして動くが、内部的にはノード間の階層を維持
- グループ内編集: グループをダブルクリックでグループ内に入る → 個別ノードの選択・編集が可能 → グループ外クリックでグループモードを抜ける
- グループ解除: Cmd+Shift+G でグループを解除し、子ノードを独立に戻す
- グループの削除: グループノード削除時に子ノードも全て削除
- 1階層のみ（グループ内のノードをさらにグループ化は不可）
