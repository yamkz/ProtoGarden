# TASK: ProtoGarden Canvas Tool
日時: 2026-03-16

## ROLE
あなたはElectronアプリ開発に精通したシニアフロントエンドエンジニアとしてふるまってください。Canvas系UIツール（Miro, Figma等）の実装パターンに詳しく、パフォーマンスとUXの両立を重視します。

## CORE GOAL
- UIプロトタイプ・ワイヤーフレーム・実装済み画面を自由なキャンバス上に並べて比較できるMacアプリ「ProtoGarden」を開発する
- Electronで.appとしてビルドし、ダブルクリックで起動可能にする
- ターミナル知識ゼロのユーザーでも使えるUXを実現する

## PURPOSE
- ユーザーはUI/UXデザインの比較を日常的に行っており、複数のプロトタイプやワイヤーフレーム、実装済み画面を一つの画面上で自由に並べて視覚的に比較したい
- 既存ツール（ブラウザタブ切替等）では比較が煩雑で、一覧性が足りない
- スクリーンショットではなく、実際に操作可能な状態で比較したいというニーズがある

## HARD REQUIREMENTS
- macOS向けElectronアプリとして動作すること
- ローカルHTMLファイル（同階層ファイル含む）の取り込み・表示が可能なこと
- 外部URLのiframe表示が可能なこと（ブロックされるサイトがあるのは許容）
- 画像のCmd+Vペースト、ドラッグ&ドロップ、ファイルピッカーによる追加が可能なこと
- テキストノードの配置・編集が可能なこと
- 全ノードがドラッグ移動・リサイズ・削除可能なこと
- 無限キャンバスでパン（Space+ドラッグ）とズーム（ホイール/ピンチ）が可能なこと
- データはユーザー指定フォルダにJSON+ファイルとして保存（外部依存なし）
- 複数ワークスペース対応（ギャラリー一覧で管理）
- 自動保存が機能すること

## DELIVERABLE
- ProtoGarden Electronアプリ一式（ソースコード + ビルド設定）
- `npm start`で開発起動、`npm run build`で.app生成が可能な状態

## DELIVERABLE SPEC

### A) ワークスペース管理（ギャラリー画面）
- ワークスペース一覧をギャラリー形式で表示（名前のみ、プレビューなし）
- 新規作成（名前入力）/ 名前変更 / 削除（確認ダイアログあり）
- ワークスペースクリックでキャンバス画面へ遷移

### B) 無限キャンバス
- CSS transformベースのパン・ズーム
- パン: Space+左クリックドラッグ / 中ボタンドラッグ
- ズーム: スクロールホイール / ピンチ（0.1x〜5x）、カーソル中心でズーム
- ズーム率表示（クリックで100%リセット）
- ビューポート状態（panX, panY, zoom）の保存・復元

### C) ノードシステム（4種類）
- **テキストノード**: キャンバスダブルクリックで作成、contenteditable、インライン編集
- **画像ノード**: Cmd+Vペースト / D&D / ファイルピッカーで追加、ファイルとして保存（base64不使用）
- **HTMLノード**: ファイルピッカーで選択→親ディレクトリごとコピー→proto-garden://プロトコルでiframe配信→インタラクト可能
- **URLノード**: URL入力→iframeで表示、インタラクト可能

### D) 全ノード共通機能
- ドラッグで自由移動
- 四隅+四辺のリサイズハンドル
- 削除ボタン（Undo対応）
- Z-indexレイヤー管理（右クリック: 最前面/最背面）
- iframe系ノードのインタラクトモード切替（ダブルクリックでON、外クリックでOFF）

### E) データ永続化
- 初回起動時にストレージフォルダをユーザーが選択
- 各ワークスペースは個別JSONファイルとして保存
- HTML・画像ファイルはassets/配下にコピーして保持
- 変更後500msデバウンスで自動保存（アトミック書き込み）
- ストレージフォルダ消失時のグレースフルなエラーハンドリング

### F) ビルド・配布
- electron-builderによるmacOS .appバンドル生成
- ダブルクリック起動、ターミナル不要
