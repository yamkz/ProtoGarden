# TASK: キャンバスUX改善・基盤整備
日時: 2026-03-17

## ROLE
あなたはElectronアプリ開発に精通したシニアフロントエンドエンジニアとしてふるまってください。Figma・Illustrator等のキャンバス系UIツールの操作体験に詳しく、スナップ・整列・履歴管理などのインタラクション実装パターンを熟知しています。

## CORE GOAL
- ProtoGardenのキャンバス操作体験を大幅に向上させる
- ノード整列・複数操作・Undo/Redoなど、キャンバスエディタとして必須のUX基盤を整備する
- ワークスペースの共有・複製など、プロジェクト管理機能を追加する

## PURPOSE
- 現状のキャンバスではノードの整列が手作業で非常に煩雑。スマートガイドで効率化したい
- 複数選択しても1要素しか移動できないバグがあり、作業効率が悪い
- Undo/Redoがないため操作ミスのリカバリーができず、安心して編集できない
- クライアントとワークスペースを共有して、編集結果をやり取りしたい
- ワークスペースの複製で、既存レイアウトをベースに新パターンを試したい

## HARD REQUIREMENTS
- 既存の全ノードタイプ（テキスト・画像・HTML・URL・Note）で動作すること
- スマートガイドはノード間の端（上下左右）と中心線を対象とし、吸着距離は5px
- 複数選択は一括移動・一括リサイズ・一括削除に対応すること
- Undo/Redoはノード操作（移動・リサイズ・削除・追加・テキスト編集）を対象、メモリ上管理、アプリ再起動でリセット
- Undo/Redo履歴の上限は50回分
- ワークスペース複製はアセット（画像・HTMLファイル）とスナップショットを含む完全複製
- エクスポートは.zip形式（JSON + assetsフォルダ）で1ファイルにまとめる
- インポート時、同名ワークスペースがある場合は「〇〇 (imported)」として別名作成

## DELIVERABLE
- スマートガイド、複数選択一括操作、Undo/Redo、ワークスペース複製、エクスポート/インポート機能を実装したProtoGardenの更新版

## DELIVERABLE SPEC

### A) スマートガイド（スナップ整列）
- ノードをドラッグ中、他ノードの端（top, bottom, left, right）と中心線（centerX, centerY）に対してスナップ判定を行う
- 吸着距離5px以内に入ったら、ガイドライン（赤い細線）を表示し、ドラッグ中のノード位置をスナップ先に吸着させる
- 同時に複数方向（水平・垂直）のスナップが可能
- ガイドラインはドラッグ終了時に消える
- リサイズ時にもスナップが機能する

### B) 複数選択の一括操作
- Shift+クリックによる追加選択/選択解除
- マーキー選択（既存）との組み合わせ対応
- 複数選択状態での一括ドラッグ移動（全ノードが相対位置を保ったまま移動）
- 複数選択状態での一括リサイズ（比率を保ったまま全ノードをスケール）
- 複数選択状態での一括削除（既存のDelete対応を維持）

### C) Undo/Redo履歴システム
- Cmd+Z: 直前の操作を取り消し
- Cmd+Shift+Z: 取り消した操作をやり直し
- 対象操作: ノードの追加・削除・移動・リサイズ・テキスト編集・z-index変更
- 履歴上限50件、超過時は最古のエントリから破棄
- 新しい操作を実行すると、現在位置より先のRedo履歴は破棄
- メモリ上管理、アプリ再起動でリセット

### D) ワークスペース複製
- ギャラリー画面で各ワークスペースに「複製」ボタンまたは右クリックメニュー
- 複製名は「〇〇 (copy)」
- ワークスペースJSON、assets/配下の画像・HTMLファイル、スナップショットを全て複製
- 複製後、ギャラリー一覧に即反映

### E) ワークスペースのエクスポート/インポート
- エクスポート: ギャラリー画面で各ワークスペースに「エクスポート」ボタン → 保存先ダイアログ → .zipファイル生成（ワークスペースJSON + assets/ + snapshots/）
- インポート: ギャラリー画面に「インポート」ボタン → .zipファイル選択 → 展開してストレージフォルダに配置 → ギャラリーに反映
- 同名ワークスペースが存在する場合は「〇〇 (imported)」として作成
- 破損したzipや不正な形式にはエラーメッセージを表示
